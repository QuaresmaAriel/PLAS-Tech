# 🚀 PLAS Tech - Sistema de Gestão de Ordens de Serviço

O **PLAS Tech** é uma solução completa para o gerenciamento de fluxos de trabalho em assistências técnicas e escritórios administrativos. Desenvolvido com foco em performance e experiência do usuário (UX), o sistema permite o controle total de ordens de serviço, agendamentos e tarefas particulares.

## 🛠️ Tecnologias Utilizadas

Este projeto foi construído utilizando tecnologias modernas de desenvolvimento web *front-end*:

* **JavaScript (ES6+):** Lógica de negócios, manipulação de DOM e arquitetura baseada em módulos.
* **Web Storage API (localStorage/sessionStorage):** Utilizada para persistência de dados no lado do cliente, garantindo uma experiência rápida e independente de backend para a versão demo.
* **SweetAlert2:** Implementação de notificações minimalistas (Toasts) para feedback em tempo real.
* **ImgBB API:** Integração com serviço externo para upload e armazenamento de evidências fotográficas das ordens de serviço.
* **CSS3 Custom Properties:** Gerenciamento dinâmico de temas (Dark/Light Mode) e interface responsiva.

---

## ✨ Principais Funcionalidades

* **Dashboard Inteligente:** Visualização clara da fila de atendimentos e tarefas pendentes.
* **Gestão de O.S.:** Cadastro detalhado com número de OS, descrição de problemas e orçamentos.
* **Linha do Tempo (Timeline):** Histórico completo de cada serviço, registrando mudanças de status, observações e fotos.
* **Upload de Imagens:** Possibilidade de anexar fotos diretamente da câmera ou galeria para documentar a entrada e saída de aparelhos.
* **Sistema de Filtros:** Busca rápida por cliente, número de OS ou tipo de atividade.
* **Personalização:** Suporte a modo escuro e ajuste dinâmico de acessibilidade.

---

## 🏗️ Arquitetura e Diferenciais Técnicos

Para esta versão de portfólio, optei por substituir o Firebase pelo **LocalStorage**. Isso demonstra:
1.  **Conhecimento em Persistência de Dados:** Capacidade de gerenciar estados e dados complexos no navegador.
2.  **Foco em UX:** O recrutador pode testar todas as funcionalidades instantaneamente, sem processos de configuração ou latência de rede.
3.  **Modularização:** O código foi estruturado de forma que a transição para um banco de dados real (como Firestore) seja feita apenas alterando as camadas de serviço.
